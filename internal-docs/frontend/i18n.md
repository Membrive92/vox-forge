# Sistema de Internacionalizacion (i18n)

## Diseno

El sistema de i18n esta basado en tipos de TypeScript. No usa ninguna libreria externa.

### Fuente de verdad: es.ts

```typescript
// src/i18n/es.ts
export const es = {
  appName: "VoxForge",
  generate: "Generar Audio",
  engineXttsV2: "Voz clonada (XTTS v2)",
  // ... 60+ keys
} as const;

export type TranslationKey = keyof typeof es;
```

`TranslationKey` se infiere automaticamente de las claves del objeto `es`. Cualquier key nueva que se anade aqui **debe** anadirse en `en.ts` tambien.

### Derivado tipado: en.ts

```typescript
// src/i18n/en.ts
import type { TranslationKey } from "./es";

export const en: Record<TranslationKey, string> = {
  appName: "VoxForge",
  generate: "Generate Audio",
  engineXttsV2: "Cloned voice (XTTS v2)",
  // ...
};
```

`Record<TranslationKey, string>` garantiza que:
- Todas las claves de `es.ts` estan presentes en `en.ts`
- No hay claves extra en `en.ts` que no existan en `es.ts`
- Si se anade una clave en `es.ts` y no en `en.ts` → **error de compilacion**

### Accessor: index.ts

```typescript
export type Translations = Record<TranslationKey, string>;

const DICT: Record<Language, Translations> = { es, en };

export function getTranslations(lang: Language): Translations {
  return DICT[lang];
}
```

### Uso en componentes

```typescript
// En App.tsx
const t = getTranslations(lang);

// En cualquier componente (recibe t como prop)
<button>{t.generate}</button>
<span>{t.engineXttsV2}</span>
```

## Cambio de idioma

```
Header → boton [ES ▼] / [EN ▼]
  │
  └─ handleToggleLang()
     ├─ lang === "es" ? setLang("en") : setLang("es")
     └─ setSelectedVoice(VOICES[newLang][0].id)
        └─ Cambia a la primera voz del nuevo idioma
```

El cambio es instantaneo — no hay recarga. Toda la UI se re-renderiza con las nuevas traducciones.

## Test de paridad

```typescript
// src/i18n/i18n.test.ts
it("es and en have the same keys", () => {
  const esKeys = Object.keys(es).sort();
  const enKeys = Object.keys(en).sort();
  expect(esKeys).toEqual(enKeys);
});

it("no translation value is empty", () => {
  for (const [key, value] of Object.entries(es))
    expect(value).not.toBe("");
  for (const [key, value] of Object.entries(en))
    expect(value).not.toBe("");
});
```

Esto captura:
- Claves que faltan en un idioma
- Valores vacios que se olvidaron de traducir

## Anadir una nueva traduccion

1. Anade la clave en `src/i18n/es.ts`:
   ```typescript
   newFeature: "Nueva funcionalidad",
   ```

2. `npm run typecheck` falla con:
   ```
   Property 'newFeature' is missing in type...
   ```

3. Anade en `src/i18n/en.ts`:
   ```typescript
   newFeature: "New feature",
   ```

4. Usa en componente:
   ```typescript
   <span>{t.newFeature}</span>
   ```

No es posible olvidar una traduccion — TypeScript lo detecta en compilacion.
